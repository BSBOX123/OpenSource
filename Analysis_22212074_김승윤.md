# Custom Reward Application - Analysis Document

## 1. Introduction

### 1) Executive Summary
현대인들은 목표 달성을 위해 다양한 계획을 세우지만, 지속적인 동기부여의 부재로 인해 실천에 어려움을 겪는다. 본 시스템은 사용자가 일상적인 목표 달성(시스템 내 업적)을 기록하여 가상의 재화를 획득하고, 이를 통해 자신이 미리 설정해 둔 맞춤형 현실 보상을 교환할 수 있도록 지원하는 개인화된 동기부여 애플리케이션이다.

### 2) Business Goals
사용자의 지속적인 목표 달성과 생산성 향상을 유도하는 것이 핵심 목적이다. 시스템이 일방적으로 제공하는 획일화된 보상이 아닌, 사용자 개인의 선호도와 가치가 반영된 맞춤형 보상 상점을 직접 구성하게 함으로써, 애플리케이션의 지속적인 사용과 현실에서의 실천에 대한 강력한 동기를 부여한다.

### 3) Technical Goals
본 프로젝트는 단독으로 진행하는 개인 과제로서 시스템의 경량화와 효율성에 초점을 맞춘다. 사용자의 정보, 업적 달성 이력, 재화 변동 내역, 보상 목록 등의 데이터는 파일 입출력 또는 경량화된 로컬 데이터베이스를 통해 관리한다. 사용자가 업적을 등록하거나 보상을 교환할 때 즉각적으로 재화가 계산 및 반영되도록 트랜잭션 로직을 설계하며, 직관적인 GUI를 제공하여 누구나 쉽게 보상을 설정하고 관리할 수 있도록 한다.

## 2. Use Case Analysis

### 2.1. Use Case Diagram (Textual Representation)
* **Actor:** User (사용자)
* **Use Cases:**
    * **Register member (회원가입):** 시스템 사용을 위한 초기 등록
    * **Log-in (로그인):** 시스템 접근 권한 획득
    * **Manage Custom Reward (맞춤형 보상 관리):** 목표 보상 추가, 수정, 삭제
    * **Record Achievement (업적 기록):** 수행한 활동을 기록하고 재화 획득
    * **Purchase Reward (보상 교환):** 누적된 재화를 소모하여 맞춤형 보상 획득
    * **Inquiry Balance & History (재화 및 이력 조회):** 현재 보유 재화 및 획득/소모 내역 확인

### 2.2. Use Case Description

### **Use Case Description: [유스케이스 이름]**

* **Use Case #1: Manage Custom Reward**
    * **Primary Actor:** User
    * **Preconditions:** 시스템에 로그인되어 있어야 함.
    * **Trigger:** 보상 상점(Reward Shop) 메뉴에서 '새 보상 추가' 또는 '수정' 버튼을 클릭했을 때.
    * **Success Post Condition:** 사용자가 원하는 현실 보상의 이름과 필요 재화량이 시스템에 정상적으로 저장된다.
    * **Main Success Scenario:**
        1. 사용자가 보상 관리 화면으로 진입한다.
        2. 새로운 보상의 이름(예: PC 하드웨어 업그레이드)과 목표 재화량(예: 50,000 포인트)을 입력한다.
        3. 저장 버튼을 누른다.
        4. 시스템은 입력값을 검증하고 보상 목록(Database)에 업데이트한다.

* **Use Case #2: Record Achievement**
    * **Primary Actor:** User
    * **Preconditions:** 시스템에 로그인되어 있어야 함.
    * **Trigger:** 메인 화면에서 '업적 기록하기' 버튼을 클릭했을 때.
    * **Success Post Condition:** 업적 내용이 기록되고, 시스템 내부 알고리즘에 의해 산정된 재화가 사용자의 총 보유 재화에 합산된다.
    * **Main Success Scenario:**
        1. 사용자가 오늘 달성한 업적(예: 전공 과제 완료)을 입력한다.
        2. 달성 난이도 또는 소요 시간을 선택한다.
        3. 등록 버튼을 누른다.
        4. 시스템은 난이도에 비례하여 획득 재화를 계산한다.
        5. 사용자의 현재 보유 재화를 갱신하고 성공 메시지를 출력한다.

* **Use Case #3: Purchase Reward**
    * **Primary Actor:** User
    * **Preconditions:** 시스템에 로그인되어 있으며, 선택한 보상의 요구치보다 보유 재화가 같거나 많아야 함.
    * **Trigger:** 보상 상점에서 특정 보상의 '교환(Purchase)' 버튼을 클릭했을 때.
    * **Success Post Condition:** 사용자의 보유 재화가 차감되고, 보상 획득 완료 내역이 기록된다.
    * **Failed Post Condition:** 보유 재화가 부족할 경우 교환이 거부되고 경고 메시지가 출력된다.

## 3. Domain Analysis

시스템을 구성하는 핵심 도메인 클래스 및 역할은 다음과 같다.

* **User:** 사용자의 고유 식별 정보(ID, Password)와 현재까지 누적된 '보유 재화(Current Balance)' 속성을 가진다.
* **RewardItem:** 사용자가 직접 등록한 맞춤형 보상 객체이다. 보상명(Name), 필요 재화량(Required_Point), 달성 여부(Status) 속성을 가진다.
* **Achievement:** 사용자가 기록한 활동 내역 객체이다. 활동 내용(Description), 획득한 재화량(Earned_Point), 등록 일자(Date) 속성을 가진다.
* **TransactionManager:** 시스템 내 재화의 흐름을 통제하는 컨트롤러 클래스이다. `Record Achievement` 실행 시 User의 재화를 증가시키고, `Purchase Reward` 실행 시 요구 재화와 현재 재화를 비교 검증(Validation)한 후 차감 로직을 수행한다.

## 4. User Interface Prototype

### 4.1. Log-in / Registration
* ID와 Password를 입력하는 텍스트 필드와 확인 버튼이 존재한다. 초기 사용자를 위한 회원가입(Register) 버튼이 하단에 배치된다.

### 4.2. Main Dashboard (Home)
* 화면 상단에 사용자의 현재 보유 재화가 크게 표시된다.
* 중앙에는 최근 1주일간 달성한 업적 리스트가 간략히 출력된다.
* 하단에는 목표 달성률이 가장 높은 추천 보상이 프로그레스 바 형태로 시각화되어 동기를 부여한다.

### 4.3. Reward Shop (보상 상점)
* 사용자가 미리 등록해 둔 맞춤형 현실 보상들이 카드(Card) 형태로 나열된다.
* 각 카드에는 보상 이름과 필요 재화량이 적혀 있으며, 보유 재화가 충분할 경우 활성화되는 교환 버튼이 포함된다. 상단에는 새로운 보상을 추가할 수 있는 보상 등록 버튼이 위치한다.

## 5. Glossary

| Terms | Description |
| :--- | :--- |
| **Custom Reward (맞춤형 보상)** | 시스템이 기본적으로 제공하는 보상이 아닌, 사용자가 자신의 동기부여를 위해 직접 명칭과 가치(필요 재화)를 설정한 현실 세계의 보상 아이템. |
| **System Currency (시스템 재화)** | 시스템 내에서 사용되는 가상의 포인트. 사용자가 긍정적인 행동(업적)을 기록할 때마다 시스템 알고리즘에 의해 산정되어 지급된다. |
| **Achievement (업적)** | 사용자가 현실에서 수행한 과제, 운동, 학습 등의 목표 달성 행위를 의미하며, 재화 획득의 기준이 된다. |
| **Transaction (트랜잭션)** | 재화가 증가(업적 달성)하거나 감소(보상 교환)하는 모든 데이터 변동 작업을 의미한다. |
